Tags Vs attributes for the listeners


When we use listeners we often use the attribute in the tag(actionListener, valueChangeListener...), but sometimes if one component needs more than one listener of the same type, we will need to use tags instead of attributes.

To be able to use tags (f:actionListener, f:valueChangedListener)so we can have more listener executed for one component:
1-We need to create our own listener implementation classes!
2-Then we add the listeners to the component using the desired listener tag.
3-We need to make sure the type attribute of the listener tag correctly points to the implementation of the listener Note:The listeners will be executed in the same order as they are called
